# 001: Java 21 verwenden

## Status

Akzeptiert.

## Kontext

Das Projekt soll eine moderne, langfristig wartbare Java-Codebasis fuer kleine
bis mittlere Anwendungen bieten. Das Domaenenmodell soll ohne unnoetige
Framework-Abhaengigkeiten auskommen und moderne Sprachmittel nutzen koennen.

## Entscheidung

Das Projekt verwendet Java 21 als Sprach- und Compiler-Ziel.

## Begruendung

- Java 21 ist eine LTS-Version.
- Records eignen sich gut fuer immutable Domaenen- und Value-Objekte.
- Moderne Sprachfeatures reduzieren Boilerplate ohne Lombok.
- Die Version passt zu einem langfristig wartbaren Projektfundament.

## Auswirkungen

- Lokale Entwicklung und CI benoetigen ein JDK 21.
- Maven wird mit `maven.compiler.release` auf `21` konfiguriert.
- Neue Sprachfeatures duerfen eingesetzt werden, wenn sie die Lesbarkeit verbessern.
